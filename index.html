The SmartClicker




Project Goals

  • Implementation of Polymorphism to truly understand the benefit

  • Have multiple inner classes in one class

  • Understand the basics of multithreading

  • Output a sessions data to a text file (I/O)

  • Gain knowledge regarding the uses of all access modifiers (default and protected especially)

Obstacles I Faced

Button not terminating loop

At the time, the program had only two buttons; one for starting the autoclicker and one for ending it. I initially had no multithreading implemented at this point; so when the program began running, it was stuck in an infinite loop once the start button was clicked.

THIS stackoverflow thread confirmed my suspicion. The fact that the program was stuck in a loop, while not being multi-threaded, meant that the event dispatching thread would not be activated. To combat this, I simply created another thread and added my infinite loop which clicks there. This solved my issue and also ensured that I didn't modify the the GUI, as Swing is not thread safe.

Making specific key stop program

I am currently having difficulty finding a way to end the program via pressing a key, while the program is not focused. Apparently this is not possible, however I'm still researching this.
